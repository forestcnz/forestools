一、功能概述

ZTools 是一个跨平台（macOS / Windows / Linux）的开源应用启动器和插件平台（类似 Alfred / Raycast / uTools）。

快速打开应用 是其核心功能：通过全局快捷键唤出搜索框，输入应用名称（支持拼音、缩写、模糊匹配）快速搜索并启动已安装的应用程序、系统设置、插件功能等。

技术栈：Electron 41Vue 3 + TSPiniaLMDBFuse.jspinyin-pro

2.2 Windows 扫描逻辑

文件：src/main/core/commandScanner/windowsScanner.ts (~13.7KB)

  • 本地化显示名称：读取 desktop.ini 的 [LocalizedFileNames] 段，支持 UTF-16LE / UTF-8；MUI 引用（@dll,-id 格式）通过原生 C++ 模块 MuiResolver 解析，解决"File Explorer.lnk" → "文件资源管理器" 的中文名问题
  • 快捷方式解析：遍历 .lnk 和 .url 文件，用 shell.readShortcutLink() 解析目标路径；.url 中的协议链接（如 steam://）保留，普通网页跳过
  • 过滤：正则过滤卸载程序、帮助文档等（/^uninstall|卸载$|help|readme|documentation|license/i）
  • 去重：按 名称|目标路径 组合去重
  • 图标：ztools-icon:// 协议 + 原生 IconExtractor 实时提取

2.3 macOS 扫描逻辑

文件：src/main/core/commandScanner/macScanner.ts (~13.2KB)

  • 收集 .app bundle，遇到即停止深入（避免扫描内部 helper）
  • 解析 Info.plist 读取 CFBundleDisplayName、CFBundleName
  • 本地化名称获取：先从 .lproj/InfoPlist.strings 读取（支持 binary / XML / UTF-16 三种格式），回退到 InfoPlist.loctable（新版 macOS 系统应用）
  • 提取 APP_NAME_SYNONYM_* 键作为搜索别名
  • 英文缩写生成："Visual Studio Code" → "vsc"
  • 50 并发 pLimit 控制

2.4 UWP 应用

Windows 平台通过原生 C++ 模块 UwpManager.getUwpApps() 获取 UWP 列表，使用 uwp:AppId 前缀标识。启动时调用 UwpManager.launchUwpApp(appId)。

2.5 增量监听

使用 chokidar 监听应用目录变化：

  • Windows：轮询模式监听 .lnk 创建/删除，depth=5
  • macOS：depth=1，监听 .app 创建/删除
  • 1 秒防抖后调用 refreshAppsCache()

2.6 缓存策略

扫描结果缓存到 LMDB（键 cached-commands），带版本号 APP_CACHE_VERSION = 4。版本变更或图标协议变更时自动重新扫描。开发模式下跳过缓存直接重新扫描。

三、搜索实现

3.1 Fuse.js 配置

文件：src/renderer/src/stores/commandDataStore.ts

new Fuse(commands, {
  keys: [
    { name: 'name', weight: 2 },         // 名称，权重最高
    { name: 'pinyin', weight: 1.5 },     // 拼音全拼
    { name: 'pinyinAbbr', weight: 1 },   // 拼音首字母
    { name: 'acronym', weight: 1.5 },    // 英文首字母缩写
    { name: 'aliases', weight: 1.5 }     // 别名
  ],
  threshold: 0,          // 精确匹配
  ignoreLocation: true,  // 不考虑距离
  includeScore: true,
  includeMatches: true
})

关键 threshold: 0 禁用 Fuse.js 内置模糊容错。模糊能力由拼音 + 缩写字段提供：输入 wx 匹配拼音首字母，weixin 匹配拼音全拼。

3.2 拼音搜索

依赖 pinyin-pro ^3.24.2，指令加载时预计算拼音字段：

import { pinyin } from 'pinyin-pro'

function toPinyinFields(name) {
  const full = pinyin(name, { toneType: 'none', type: 'string' })
    .replace(/\s+/g, '').toLowerCase()    // "微信" → "weixin"
  const abbr = pinyin(name, { pattern: 'first', toneType: 'none', type: 'string' })
    .replace(/\s+/g, '').toLowerCase()    // "微信" → "wx"
  return { pinyin: full, pinyinAbbr: abbr }
}

预计算而非搜索时实时转换，保证搜索性能。

3.3 英文缩写提取

文件：src/main/utils/common.ts

function extractAcronym(name: string): string {
  // 空格分隔："Visual Studio Code" → "vsc"
  const words = name.split(/\s+/).filter(Boolean)
  if (words.length > 1) return words.map(w => w[0].toLowerCase()).join('')
  // 驼峰命名："VisualStudioCode" → "vsc"
  const capitals = name.match(/[A-Z]/g)
  if (capitals && capitals.length > 1) return capitals.map(c => c.toLowerCase()).join('')
  return ''
}

五、交互流程

全局快捷键

默认 Option+Z（macOS）/ Alt+Z（Windows）。支持双击修饰键模式（Ctrl+Ctrl、Alt+Alt），通过 doubleTapManager + uiohook-napi 实现。多显示器时自动跟随鼠标所在屏幕。

窗口行为

  • 无边框、透明、总在最前
  • blur 自动隐藏（支持拖拽文件防护逻辑）
  • 可配置自动回到搜索页的延时

搜索交互流

快捷键唤起 自动粘贴（可选） 输入搜索 ↑↓ 选择 Enter 确认 启动应用 Esc 分级退出

Esc 退出的分级逻辑：清输入框 → 清粘贴内容 → 隐藏窗口

右键菜单

  • 从使用记录删除 / 打开文件位置 / 以管理员身份运行（Win）
  • 固定到搜索框 / 取消固定
  • 固定到超级面板 / 取消固定
  • 插件设置 / 详情 / 全局快捷键

八、架构特点

  1. 万物皆指令：应用、插件、系统设置统一为 Command 类型
  2. 双层搜索：Fuse.js 模糊搜索 + 正则指令匹配
  3. 拼音预计算：指令加载时生成拼音字段，搜索时零开销
  4. 平台原生集成：Windows 的 desktop.ini/MUI 解析、macOS 的 lproj/loctable
  5. 增量更新：chokidar 监听 + 1s 防抖 + 增量刷新
  6. 完整生命周期：扫描 → 缓存 → 搜索 → 启动 → 历史 → 统计 → 偏好
  7. 高性能设计：LMDB 持久化、LRU 图标缓存、串行原生调用
