一、概览

ZTools 是一个基于 Electron 41 + Vue 3 + TypeScript + LMDB 的跨平台应用启动器和插件平台，类似 uTools / Raycast / Alfred 的开源实现。

插件系统是其核心能力，支持UI 插件（有界面）和无界面插件（后台运行）两种形态，提供从开发到分发的完整工具链。

ztools-api-types 子模块提供 TypeScript 类型定义（NPM 发布），ztools-plugin-cli 子模块提供脚手架与发布 CLI。

三、插件生命周期

由 pluginManager.ts + pluginAssemblyCoordinator.ts + resources/preload.js 三方协作完成。

装配状态机

状态路径：idle → assembling → domReady → readyToDisplay → displayed → aborted

详细流程

搜索到插件指令 beginAssembly createPluginView 读 plugin.json 创建隔离 Session 创建 WebContentsView 注册 preload + loadURL dom-ready processPluginMode 派发 PluginEnter 用户交互 派发 PluginOut 或 kill

关键设计

  • 粘性进入事件缓存（pendingEnterPayload）：如果 onPluginEnter 还没注册就已收到进入事件，缓存 payload 等待回调注册后回放
  • 事件串行化：lifecycleChains Map 按 webContents id 串行化，防止 PluginOut / PluginEnter 交错
  • 重入防抖（shouldSkipReEnter）：同一插件、同一指令、同一文本的连续进入直接跳过

四、插件类型

4.1 UI 插件

plugin.json 包含 main 字段，指向 HTML 入口。ZTools 创建 WebContentsView 加载该 URL，通过 window.ztools 调用 API。

  • 可设置子输入框（setSubInput）
  • 可调整自身高度（setExpendHeight）
  • 支持 list 模式（搜索结果列表在插件内部渲染）
  • 支持 mainHide 模式（不弹出主窗口）
  • 可分离为独立窗口

示例：internal-plugins/setting/（Vue 3 + Vite + UnoCSS）

4.2 无界面插件（Headless）

plugin.json 没有 main 字段。加载隐藏空白 HTML，通过 preload.js 的 window.exports 机制执行：

window.exports = {
  process: {
    mode: 'none',
    args: {
      enter: async (action) => {
        // 处理逻辑，无 UI
      }
    }
  }
}

示例：internal-plugins/system/（系统指令：关机、重启、截图等）

4.3 内置插件

存放在 internal-plugins/ 目录，定义在 internalPlugins.ts：

export const BUNDLED_INTERNAL_PLUGIN_NAMES = ['setting', 'system']
  • 特点 不经过市场安装/卸载流程
  • 特点 有更高 API 权限（window.ztools.internal 命名空间）
  • 特点 生产环境使用本地 HTTP server 加载（绕过 file:// CSP 限制）
  • 特点 不可被禁用

4.4 开发版插件

开发版插件名自动追加 __dev 后缀（定义在 pluginRuntimeNamespace.ts）：

export const DEV_PLUGIN_SUFFIX = '__dev'

在数据库命名空间、Session 隔离等所有层面与安装版完全独立，可同时存在。

6.3 API 分发器机制

通过 pluginApiDispatcher.ts 统一注册，支持三种 IPC 模式：

  • 同步：插件端用 ipcSendSync 调用，主进程 event.returnValue
  • 异步：插件端用 ipcInvoke 调用，主进程返回 Promise
  • 推送：插件端用 ipcSend 调用（单向）

安装流程

用户点安装 下载 .zpx 文件 extractZpx 解压 校验 plugin.json 写入 LMDB 通知刷新

ZPX 包格式

gzip(asar archive) 或 brotli(asar archive)，实现在 zpxArchive.ts。

升级策略

  • 语义化版本号对比
  • 先卸载旧版本，再安装新版本
  • 升级时保留插件数据（deleteData: false）

九、数据隔离

LMDB 命名空间

ZTOOLS/               → 主程序数据
PLUGIN/{name}/        → 插件数据（含 __dev 后缀则完全隔离）
SYNC/                 → 同步数据

Session 隔离

persist:{pluginName}  → 插件主视图 Session
{pluginName}.zbrowser → zbrowser 专用 Session

隔离层级

  • 数据库前缀：PLUGIN/my-plugin/ vs PLUGIN/my-plugin__dev/
  • Session partition：persist:my-plugin vs persist:my-plugin__dev
  • 窗口尺寸存储：独立键名
  • 数据清理：删除插件时默认清除全部数据，可保留供重新安装

十二、开发流程

ztools create my-plugin 选择模板 编写 plugin.json + preload ZTools 设置添加开发中插件 热重载开发 ztools publish 发布
  • 开发版自动追加 __dev 后缀，数据完全隔离
  • 可同时安装同一插件的开发版和正式版
  • 内置 ztools-developer-plugin 用于调试
