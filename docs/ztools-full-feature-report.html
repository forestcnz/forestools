一、项目定位与总览

ZTools 是 uTools 的开源实现，定位为高性能、可扩展的跨平台应用启动器与插件平台（类似 Alfred / Raycast）。它把"快速启动 + 万物皆指令 + 插件生态 + AI/自动化能力"打包成一个统一的桌面入口。

核心特性一句话概括：

  • 入口 拼音搜索 + 正则 + 历史 + 固定的快速启动
  • 生态 UI 插件 / 无界面插件 / 内置插件，完整 API 与市场分发
  • 效率 剪贴板管理、超级面板、悬浮球、网页快开、系统指令
  • 智能 AI 对话、MCP Server、ZBrowser 浏览器自动化、离线翻译
  • 同步 WebDAV 云同步、应用内更新、独立 updater 替换 asar
  • 性能 LMDB 键值存储、WebContentsView 架构、C++ 原生模块

当前路线图中已完成：快速启动、插件系统、剪贴板、跨平台、LMDB、主题、数据隔离、插件市场、全局快捷键自定义、插件分离窗口。规划中：插件自动更新、云同步（已部分实现 WebDAV）、Linux 支持、MCP 工具集（已实现）。

1.2 进程架构

ZTools 是标准的 Electron 三段式架构，但插件运行在独立的隔离 Session + WebContentsView中，而非主窗口内的 iframe/Webview，这是其高性能与安全隔离的关键。

进程职责关键文件
主进程
src/main		 窗口管理、插件生命周期、数据库、原生模块、所有 API handler、同步/AI/MCP/翻译/截图核心 managers/（window / plugin / clipboard / assembly）、api/、core/
主程序渲染
src/renderer		 搜索界面、超级面板 UI、更新窗口；两种视图：Search / Plugin App.vue、stores/、components/
插件渲染 每个插件独立 WebContentsView + 独立 Session，加载 resources/preload.js resources/preload.js（不经 Vite 构建，原生 JS）

主进程内部又有清晰的分层：managers/（生命周期管理器，windowManager 1133 行、pluginManager 1882 行、clipboardManager 860 行）→ api/（按消费者分 shared / renderer / plugin 三套 IPC）→ core/（领域引擎：lmdb / sync / zbrowser / commandScanner / native 等）。

二、快速启动与搜索

ZTools 的核心入口。默认快捷键 Option+Z（macOS）/ Alt+Z（Windows）唤起主窗口。

万物皆指令（Command）

所有可搜索内容统一为 Command 类型：

  • type: 'direct' — 直接启动：subType: 'app'（系统应用）/ subType: 'system-setting'（Windows 设置）
  • type: 'plugin' — 插件功能
  • type: 'builtin' — 内置系统指令（截图、取色等）

指令扫描

跨平台扫描系统已安装应用（core/commandScanner/）：

  • macOS：解析 .app 的 Info.plist
  • Windows：扫描开始菜单 .lnk 快捷方式
  • Linux：linuxScanner.ts（实验性）

扫描结果缓存，非必要不重扫，提升启动与切换性能（v2.4.0 优化）。启动由 core/commandLauncher/ 按平台分发。

搜索引擎

  • Fuse.js 模糊匹配 + pinyin-pro 拼音搜索（全拼/首字母）
  • 历史记录、固定应用（Pin）、聚合视图（历史 + 固定 + 窗口匹配 + mainPush）
  • 指令别名（v2.4.0 新增，支持多触发词，匹配结果高亮）
  • Tab 键在搜索结果间切换（v2.4.0）
  • 本地启动项（localShortcuts）：自定义文件/文件夹/应用快捷方式，搜索时显示安装路径

唤醒控制

全局快捷键唤醒（支持 F1–F12 功能键、双击修饰键检测 doubleTapManager.ts）、唤醒黑名单（屏蔽特定应用触发，v2.4.0）、应用快捷键（支持携带参数）、主窗口记忆位置并恢复。

三、剪贴板管理

跨平台原生实现的剪贴板历史管理（managers/clipboardManager.ts 860 行 + C++ ClipboardMonitor）。

  • 历史记录：文本 / 图片 / 文件，支持分页、搜索、过滤
  • 去重：修复某些软件触发多次重复记录（v2.4.0）
  • 可配置历史数据保存天数（v2.4.0）
  • 写入回贴：可选写入后自动粘贴
  • 插件可读写：通过 ztools.clipboard 与 copyText/copyImage/copyFile API

插件剪贴板 API

ztools.clipboard.getHistory(page, pageSize, filter)
ztools.clipboard.search(keyword)
ztools.clipboard.write(id, shouldPaste)
ztools.clipboard.writeContent({ type:'text'|'image', content }, shouldPaste)
ztools.clipboard.onChange(callback)   // 监听变化
ztools.copyText(text) | copyImage(img) | copyFile(path) | getCopyedFiles()

四、超级面板

类 uTools 的智能右键面板（core/superPanelManager.ts + components/SuperPanel.vue）。

  • 触发方式：鼠标中键 / 长按右键
  • 智能识别：自动识别剪贴板内容（文本/图片/文件/链接），匹配可用插件
  • 紧凑布局（v2.4.0），节省屏幕空间
  • 固定指令到面板（pinToSuperPanel）、屏蔽特定应用（blockedApps）
  • 集成离线翻译（选中翻译，Windows 已修复初始化问题）
  • 图片分析（analyzeImage，基于 sharp）

5.5 数据隔离与安全

  • 数据隔离：插件数据库自动加 PLUGIN/{pluginName}/ 前缀，主程序用 ZTOOLS/
  • Session 隔离：每插件独立 Session（含独立 zbrowser 窗口池）
  • 清理：删除插件时自动清理其历史、固定列表、数据（v2.4.0 修复）
  • API 分发：新 API 走 pluginApiDispatcher.ts 注册表，统一 plugin.api 通道，三方法 ipcSendSync/ipcInvoke/ipcSend
  • 插件可被禁用而无需卸载（v2.4.0）

5.6 插件市场与分发

  • 插件托管于 ZTools-plugins GitHub Releases，plugins.json 描述列表
  • 包格式：ZIP（含 plugin.json）；生产部署用 ZPX（gzip asar，utils/zpxArchive.ts）
  • 一键安装 / 升级（语义化版本比较，先卸旧再装新）/ 详情查看
  • 支持从 npm 安装、从本地文件导入、开发版导入
  • 配套子模块：ztools-api-types（TS 类型，NPM 发布）、ztools-plugin-cli（脚手架与发布 CLI）

六、AI 集成

OpenAI 兼容格式的 AI 能力（api/plugin/ai.ts + api/renderer/aiModels.ts），依赖 openai SDK。

  • 模型管理：用户在设置中配置多个 OpenAI 兼容模型（baseURL + key + 模型名）
  • 插件调用：ztools.ai(option, streamCallback)，支持流式与非流式，返回带 .abort() 的 PromiseLike
  • 模型列表：ztools.allAiModels() 获取所有可用模型
  • 内置插件可管理模型 CRUD（internal.aiModels）
  • 图像分析 analyzeImage（sharp 处理）

七、MCP Server

ZTools 把自身及所有插件声明的工具暴露为 MCP (Model Context Protocol) JSON-RPC 服务（core/mcpServer.ts + api/plugin/tools.ts），让外部 AI 客户端可调用本地能力。

  • 插件通过 ztools.registerTool(name, handler) 声明工具
  • 主进程汇总所有已装插件的工具，通过 MCP 协议对外暴露
  • 工具名生成支持插件名称格式化（v2.4.0）
  • 设置中可配置端口、密钥、启停；mcpServerStatus / mcpServerTools 查询状态与工具列表

意义 ZTools 既是 AI 的"桌面入口"，又能反过来成为 AI Agent 的工具提供方——这是其区别于普通启动器的关键定位。

八、ZBrowser 浏览器自动化

uTools ubrowser 的实现（core/zbrowser/ + api/plugin/zbrowser.ts + resources/zbrowser/ runner/client）。

  • 窗口池管理：zbrowserManager.ts 维护每插件独立 Session 的浏览器窗口池，复用 getIdleUBrowsers()
  • 操作队列执行器：zbrowserExecutor.ts 串行执行链式操作
  • 链式 Builder API：ztools.zbrowser().goto().wait().click().value()...
  • 设备预设（devices.ts）、代理设置（异步，破坏性变更：uTools 为同步）
  • 缓存清理 clearUBrowserCache

九、WebDAV 云同步

跨设备数据同步（core/sync/ + api/renderer/sync.ts，依赖 webdav）。

  • 同步引擎 syncEngine.ts + WebDAV 客户端 webdavClient.ts
  • 插件目录变更监听 pluginSyncWatcher.ts（标记脏数据）+ 哈希计算 pluginHasher.ts
  • 支持测试连接、保存配置、手动同步、强制从云端下载、停止自动同步、查询未同步数
  • v2.4.0 修复 %2F 路径保留目录结构被服务端拒绝、层级路径与目录创建异常

十、离线翻译

core/translationManager.ts，基于 Bergamot WASM + Firefox 翻译模型，完全离线、无云端依赖。集成于超级面板的"选中翻译"功能（updateSuperPanelTranslate / getTranslationStatus）。

十一、辅助功能

11.1 HTTP API

内置本地 HTTP 服务（core/httpServer.ts），允许第三方应用/脚本控制 ZTools。

  • 监听 http://127.0.0.1:36578（端口可改），Bearer Token 认证，开启 CORS
  • 接口：GET /（状态，免认证）、POST /api/window/show|hide|toggle（可选 text 填充搜索框）
  • 仅本机访问、默认关闭、请求体限 1MB、密钥重生成后旧密钥立即失效
  • 提供 JS / Python / AppleScript / PowerShell 调用示例

11.2 悬浮球

core/floatingBallManager.ts — 屏幕置顶小圆球，单击/双击触发，可配置启停与字母样式。

11.3 网页快开

自定义搜索引擎（api/renderer/webSearch.ts），支持 CRUD 与自动抓取 favicon。输入关键词直达搜索结果。

11.4 系统指令与设置

  • 系统指令（api/renderer/systemCommands.ts）：截图（内置，v2.4.0）、取色、Windows 注销/睡眠（改用 PowerShell）、macOS 系统操作
  • Windows 系统设置（core/systemSettings/ + msSettingsUris.ts）：通过 ms-settings: URI 集成，可作为指令搜索启动
  • 原生模块（C++，core/native/）：ClipboardMonitor、WindowMonitor、WindowManager、ScreenCapture、ColorPicker、MouseMonitor
  • 区域截图（Windows 原生）、屏幕截图 core/screenCapture.ts

11.5 主题定制

系统/亮色/暗色模式，6 种主题色 + 自定义；亚克力透明度可调（macOS 窗口材质 material 可选）。统一滚动条样式与字体规范（v2.4.0）。通用控件类 .btn .input .select .toggle .card 复用。

11.6 应用更新

api/updater.ts + 独立 updater 程序，绕过 Electron 常规更新限制。

检查 latest.yml 下载平台更新包 ZIP 启动独立 updater 应用退出 替换 app.asar 自动重启

更新源 GitHub Releases，命名 update-{platform}-{arch}-{version}.zip（如 update-darwin-arm64-1.2.8.zip）。updater：macOS ztools-updater、Windows ztools-agent.exe（scripts/updater.mjs 构建）。支持自动检查开关（蓝奏云备用源 utils/lanzou.ts）。

十二、数据持久化（LMDB）

全部数据落在 LMDB（core/lmdb/），路径 userData/lmdb，三个库：main / meta / attachment。

  • 命名空间：ZTOOLS/（主程序）、PLUGIN/{name}/（插件自动隔离）、SYNC/（同步配置）
  • 同时提供同步 API（syncApi.ts，供插件 sendSync 用）与异步 Promise API（promiseApi.ts）
  • 支持附件存储（postAttachment / getAttachment）
  • 启动数据迁移 startupDataMigrations.ts

性能 LMDB 是嵌入式 KV 存储中性能最高的之一，配合 WebContentsView 架构，是 ZTools"极速响应"卖点的技术基础。

十三、总结

产品定位

ZTools 不只是一个启动器，而是"启动器 + 插件平台 + AI/自动化 hub"的三合一。它以 uTools 开源替代为切入点，但在 MCP / AI / 浏览器自动化 / 离线翻译等方向上做了显著扩展，使其具备成为"本地 AI Agent 桌面入口"的潜力。

工程亮点

  • 分层清晰：managers（生命周期）→ api（按消费者三套 IPC）→ core（领域引擎）
  • 插件隔离彻底：独立 Session + WebContentsView + 数据库命名空间前缀
  • 双 preload 设计：主程序（contextBridge，热重载）vs 插件（原生 JS，需重启）
  • 统一 API 分发器 pluginApiDispatcher 替代散落的 ipcMain.handle
  • 独立 updater 替换 asar，规避 Electron 更新限制
  • 原生 C++ 模块补齐 Electron 在剪贴板监听/窗口管理/截图上的短板

当前短板 / 待完善

  • Linux 支持仍为实验性（scanner/launcher 已就位但未完整打包）
  • 插件自动更新为路线图项（市场已有升级机制，自动检查待补）
  • 同步为 WebDAV 单一方案，无通用云同步
  • resources/preload.js 不经构建，插件 API 修改需重启，开发体验略受限

对本项目（forestools）的启示

本仓库当前是一个 Tauri 应用（见 src-tauri/），已实现窗口记忆、托盘、Escape 隐藏等基础能力。ZTools 作为同类成熟产品，其万物皆指令的统一 Command 模型、插件隔离 Session 架构、统一 API 分发器、超级面板智能识别、MCP 暴露本地工具给 AI等设计，均可作为后续功能演进的参考蓝本。
